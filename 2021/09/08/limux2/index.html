Opt:安装程序文件存放的位置

三、学习Linux的指令(命令)

1.访问命令：

​ cd 文件夹名 ：访问某个文件夹

cd 文件夹1/文件夹2：访问级联目录

cd ../ :返回上一级

cd ~:返回root登陆界面

2.列出当前目录的文件与文件夹

Ls —–只列出当前目录下的文件和文件夹的名字

ll —-列出当前目录下的文件夹与文件的名字和信息

3.创建文件夹的命令

mkdir 文件夹名: 在当前的目录下，创建好该文件夹

Mkdir -p 文件夹1/文件夹2/文件3： 在当前目录下，创建文件夹1，然后文件1里面创建文件夹2，文件2里面创建文件夹3.

4.删除空文件夹

rmdir 文件夹名： 注意只能空文件夹

5.访问文件的命令：

Cat：访问的是文件的最后面的部分

More: 访问文件，它会显示文件的百分比，按下空格键进行切换。

Less: 访问文件内容：根据pgup和pgdn来进行上下翻页

tail:

tail 文件名:访问文件

tail -数字 文件名：访问后几行

6.删除的命令

A. 询问式删除： rm 文件名 （y/n）

A. 不询问式删除： rm -rf 文件名

B. 自杀式删除：

rm -rf *

rm -rf ./* : 把当前目录下面所有的文件信息删除掉

7.复制、粘贴：

cp 源文件 目标文件

8.剪切移动

Mv 源文件 目标路径(文件夹)

复制和剪切也支持级联目录的操作：

cp 1.txt ../../ :复制到上一级的上一级

mv ../aa/bb :剪切到上一级的aa目录下bb文件夹里面

9.打包压缩的指令：

压缩格式： tar -Czvf 压缩包.gz ./*

解压格式： tar -xvf 压缩包名.gz

一、Linux上面部署jdk软件

  1. 通过ftp软件把jdk的安装包上传到Linux服务器上面

​

​ (1) 要配置jdk之前准备：

​ 查询一下Linux系统自带的版本

​ 上图所示：已经查出自带的jdk版本

​ 查询全部的jdk版本：

​ rpm -qa | grep java

​ 根据版本号来卸载掉:

​ rpm -e –nodeps 版本号

​ 如何检验卸载自带的完毕：

​ 下面这条指令查询完毕，没有任何提示，则表示卸载完毕

​

安装jdk的具体步骤：

  1. 进入 usr/local/src 里面，创建一个 java的文件夹

  2. 把jdk程序直接传到这个java文件里面

  3. 解压： tar -xvf jdk-7u71-linux-i586.tar.gz

  4. 安装依赖环境： yum install glibc.i686

  5. 【进入根目录】修改配置文件： vi etc/profile

    在末尾追加下面代码：

    ​ #set java environment

    ​ JAVA_HOME=/usr/local/src/java/jdk1.7.0_71

    ​ CLASSPATH=.:$JAVA_HOME/lib.tools.jar

    ​ PATH=$JAVA_HOME/bin:$PATH

    ​ export JAVA_HOME CLASSPATH PATH

    6.加载配置文件： source etc/profile

    7.通过查询 jdk版本的指令：java -version

    完成上述7步，jdk环境配置ok,下面配置一个tomcat服务器

六、Linux搭建tomcat服务器

  1. 创建tomcat文件夹【usr/local/src/tomcat】，上传tomcat的包,解压

  2. 传tomcat的包到该文件夹里面，然后解压

    (1) sh startup.sh 开服务

    (2) sh shutdown.sh 关服务

  3. 做到这一步，服务器正常开关没问题，但是不能进行远程访问，因为需要设置防火墙。设置防火墙规则的代码：

    /sbin/iptables -I INPUT -p tcp –dport 8080 -j ACCEPT

    /etc/rc.d/init.d/iptables save
